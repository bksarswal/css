Selectors and Specificity:

Selectors and Specificity in CSS
are fundamental concepts in CSS that determine how styles are applied to HTML elements. 

Selectors

Selectors are patterns used to target specific elements in the DOM for styling.

Types of Selectors


Universal Selector (*): 
Targets all elements.
* {
  margin: 0;
}


Type Selector
: Targets all elements of a specific type (e.g., div, p).
p {
  color: blue;
}

Class Selector  (.classname) :
  Targets elements with a specific class.
.highlight {
  background-color: yellow;
}

ID Selector (#id):
 Targets an element with a specific ID.
#header {
  font-size: 24px;
}

Attribute Selector:
 Targets elements based on their attributes.
input[type="text"] {
  border: 1px solid gray;
}

Group Selector: 
Targets multiple selectors at once, separated by commas.
h1, h2, h3 {
  font-family: Arial, sans-serif;
}

Pseudo-classes:
 Targets elements based on their state.
a:hover {
  color: red;
} 

Pseudo-elements: 
 Targets specific parts of an element.
p::first-line {
  font-weight: bold;
}

Combinators: 
 Define relationships between elements:

Descendant ( ):
 Targets all descendants.
div p {
  color: green;
}

Child (>):
 Targets direct children.
div > p {
  margin: 10px;
}

Adjacent Sibling (+):
 Targets the next sibling.

h1 + p {
  color: gray;
}

General Sibling (~): 
Targets all siblings.

h1 ~ p {
  font-style: italic;
}


Specificity

Specificity determines which CSS rule is applied when multiple rules match the same element. 
It is calculated based on the types of selectors in a rule.

Specificity Calculation

Inline Styles: Add 1000 (e.g., style="color: red;").
ID Selectors: Add 100 for each #id.
Class, Attribute, and Pseudo-class Selectors: Add 10 for each (e.g., .class, [attr=value], :hover).
Type and Pseudo-element Selectors: Add 1 for each (e.g., div, p::before).
Universal Selector (*): Adds 0 and has no specificity weight.

Avoid Inline Styles and !important: Use them only when absolutely necessary to maintain clean and manageable CSS.
